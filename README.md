# SysArch Studio — システム構成図エディタ

ブラウザだけで動く、モダンな**システム構成図・システム連携図**エディタです。
ドラッグ&ドロップで部品を並べ、ノード同士を線でつなぎ、**SVG / PNG / PDF** で書き出してスライドや資料にそのまま貼り付けられます。

- 🏗 **2 種類の図に対応** — システム構成図（Web三層＋非同期）／システム連携図（部門ゾーン × SaaS × プロセス）。サンプルを 1 クリックで読み込み
- ⌨ **テキストで図を書く（Diagram as Code）** — 簡潔なテキストからワンクリックで図を生成、現在の図をテキストに逆変換。図とコードを行き来しながら作図できます（ツールバー「⌨ コード」）
- 🗂 **ゾーン / 部門枠** — タイトルヘッダー付きコンテナ。枠をドラッグすると中の部品も一緒に動く。背景バンド・タイトルバナーも
- 🔌 **サービス / ロゴタイル** — 主要 SaaS を実アイコン（[Simple Icons](https://simpleicons.org) 収録分）＋ブランド色タイルで表現。未収録サービスは頭文字タイルにフォールバック（Salesforce / Slack / Zoom / Google / GitHub ほか）
- 🖼 **SVG（ベクター・再編集可）/ PNG（2x・4x）/ PDF（1 ページ）書き出し**、PNG クリップボードコピー対応
- 🔀 **配線** — 線ごとに「なめらか（ベジェ）／直角（オルソゴナル）」を切替、矢印・ラベル・線種
- 🧲 **自動整列**（左→右レイヤー配置）・**コピー&ペースト**・Undo/Redo・ミニマップ
- 🌌 **Three.js のアンビエント背景** + Obsidian 風グラス UI、**5 種の配色テーマ**（Midnight Blueprint ほか＋ライト）
- 🎛 **依存ゼロ・完全クライアントサイド** — GitHub Pages にそのまま置くだけで動作。JSON 保存/読込・ローカル自動保存

## 使い方

| 操作 | やり方 |
|------|--------|
| 部品を置く | 左パレットから**ドラッグ**、またはクリックで中央に配置 |
| つなぐ | ノードにマウスを乗せると出る**端の○をドラッグ**して相手ノードへ |
| 選択・編集 | ノード / 線をクリック → 右のインスペクタでラベル・色・線種などを編集 |
| 移動 | ノードをドラッグ（8px グリッドにスナップ） |
| 画面移動 | 空白をドラッグ、または **Space + ドラッグ** / 中ボタンドラッグ |
| ズーム | マウスホイール（カーソル基準） |
| 全体表示 | ツールバーの倍率表示、または **F** キー |

### ショートカット
`Del` 削除 / `Ctrl+C`・`Ctrl+V` コピー&ペースト / `Ctrl+D` 複製 / `Ctrl+Z` 元に戻す / `Ctrl+Shift+Z`（`Ctrl+Y`）やり直し / `Ctrl+S` JSON保存 / `F` 全体表示 / `Esc` 選択解除

### ゾーン / 部門枠
「レイアウト / ゾーン」から **ゾーン枠・部門ボックス**を置くと、タイトルヘッダー付きのコンテナになります。**ヘッダーをドラッグ**すると枠の中に入っている部品も一緒に移動します（中身は個別にドラッグ／接続も可能）。**背景バンド**は図全体の下地、**タイトル**は見出しバナーに使えます。

### テキストで図を書く（Diagram as Code）
ツールバーの **⌨ コード** を開き、次の書式でテキストを書いて **▶ 図を生成**（`Ctrl+Enter`）すると図になります。**↩ 図をコードに** で現在のキャンバスを逆変換できます。

```text
title 受発注システム連携図
zone 営業部門 {
  crm: salesforce [CRM] | 商談 / 見積
  portal: web [受注ポータル]
}
zone 基幹 / バックオフィス {
  erp: app [基幹ERP] | 在庫 / 会計
  db: db [受注DB] | PostgreSQL
}
crm -> portal : 見積連携
portal --> erp : 受注登録
erp ..> mail : 出荷通知
crm <-> erp : マスタ同期
```

- **ノード** `id: 種別 [ラベル] | 補足`（種別・ラベル・補足は省略可）。`種別` は左パレットの英語 ID（`salesforce` / `gateway` / `db` / `queue` / `table` …）
- **ゾーン** `zone 名前 { … }` — 中に書いたノードが枠に入ります／**見出し** `title …`
- **線** `->` 実線・`-->` 破線・`..>` 点線・`<->` 双方向・`---` 矢印なし・`<-` 逆向き。ラベルは末尾に `: 文字`
- ゾーンは左から順に列として自動レイアウトされます（生成後は自由に移動・整列可能）

### 書き出し
右上の **書き出し ▾** から:
- **SVG** — ベクター。Illustrator / PowerPoint / Figma で再編集可。拡大しても劣化しない
- **PNG (2x / 4x)** — スライド貼り付け向けのラスター画像。背景色付き
- **PDF (1 ページ)** — 図をそのまま 1 ページに収めた PDF。スライド／印刷用
- **PNG コピー** — クリップボードへ直接コピー（対応ブラウザ）

> グリッドの点は書き出しには含まれません。図の要素だけがクリーンに出力されます。

## ローカルで動かす

ES Modules を使うため `file://` 直開きではなく簡易サーバー経由で開いてください。

```bash
# どれか一つ
python -m http.server 8000
npx serve .
```

ブラウザで `http://localhost:8000` を開きます。

## GitHub Pages へ公開する

1. このフォルダを GitHub リポジトリとして push
2. リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に設定
3. `main`（または `master`）に push すると `.github/workflows/pages.yml` が自動デプロイ

> `.nojekyll` を含めているので Jekyll による加工は行われません。ビルド不要でそのまま配信されます。

## 構成

```
index.html          画面レイアウト
css/style.css       グラス UI / 明暗テーマ
js/
  main.js           全体の配線（パレット・ツールバー・インスペクタ・デモ・ミニマップ）
  editor.js         SVG 編集エンジン（配置 / 接続 / 選択 / パン&ズーム / Undo / 永続化）
  export.js         自己完結 SVG 生成 + PNG ラスタライズ + クリップボード
  background.js      Three.js アンビエント背景（未ロード時は静的グラデにフォールバック）
  nodes.js          部品カタログ（種別・色・グループ）
  dsl.js            テキスト DSL ⇄ 図の相互変換（純粋関数・依存ゼロ）
  icons.js          ラインアイコン（Feather 風）
```

### 設計メモ
- ノード / 線は **純粋な SVG**（`foreignObject` 不使用）で描画。テーマ変数は描画時に**実際の色値へ解決**して属性に入れているため、書き出した単体 SVG がそのままの見た目で開き、PNG ラスタライズでも canvas が汚染されません。
- Three.js は CDN（unpkg, importmap）から読み込み。ブロックされても背景が消えるだけで編集機能はそのまま動きます。
